Application d'extraction de données

Extraction sur mesure de portions restreintes de données

← Retour à l'accueil

Contexte

Les analystes avaient souvent besoin de travailler sur des portions restreintes de données sans avoir à manipuler la masse totale des bases de laboratoires (plusieurs millions d'enregistrements), qui devenait trop volumineuse pour une analyse rapide sur des machines standards.

Problématique

  • Bases de données trop volumineuses pour une analyse fluide (> 5 Go)
  • Temps de chargement et de traitement trop longs (plusieurs minutes)
  • Besoin de travailler sur des sous-ensembles pertinents
  • Contraintes de ressources informatiques (ordinateurs modestes)
  • Nécessité de garder la traçabilité des extractions

Solution

Développement d'une application d'extraction sur mesure permettant :

1. Sélection de variables

  • Choix interactif des colonnes à extraire
  • Préservation des relations entre tables
  • Export avec métadonnées

2. Filtres avancés

  • Par région, district, période
  • Par type de laboratoire, niveau
  • Par plage de dates, indicateurs spécifiques
  • Combinaison de critères (filtres booléens)

3. Extraction optimisée

  • Requêtes optimisées pour la performance
  • Traitement par lots pour les très gros volumes
  • Cache des extractions fréquentes
  • Parallélisation des requêtes

4. Export multi-formats

  • CSV, Excel, JSON, Shapefile
  • Compression automatique des fichiers
  • Export direct vers R/Python via API

Architecture technique

  • Frontend : React.js avec interface intuitive
  • Backend : FastAPI (Python) asynchrone
  • Base de données : PostgreSQL avec optimisations
  • Cache : Redis pour les requêtes fréquentes
  • File d'attente : Celery pour les extractions longues

Technologies utilisées

Python FastAPI React PostgreSQL Redis Celery Docker
